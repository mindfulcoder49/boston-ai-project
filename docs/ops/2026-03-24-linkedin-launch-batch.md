# 2026-03-24 LinkedIn Launch Batch

## Purpose

This document contains the first recommended PublicDataWatch LinkedIn Page post batch after page creation.

These drafts follow the voice and operating rules in [social-distribution.md](./social-distribution.md).

## Publishing Rule

The first five public posts should be reviewed by the founder before publishing.

Use one direct link per post when possible.

Prefer calm, specific product language over broad civic claims.

## Post 5: How PublicDataWatch Should Be Used

Suggested slug:
- `method_note_public_records`

Suggested link:
- `https://publicdatawatch.com/?utm_source=linkedin&utm_medium=social&utm_campaign=pdw_page_launch&utm_content=method_note_public_records`

Draft:

`A method note about PublicDataWatch: public records are useful, but they are not the whole story.`

`This product is meant to make city data easier to explore. It is not meant to flatten a neighborhood into a single narrative or imply that a map view is equivalent to lived reality. Public records have delays, gaps, category quirks, and interpretation limits.`

`That is why the product direction is to make the data easier to access while staying explicit about what the data can and cannot support.`

`Main site: https://publicdatawatch.com/?utm_source=linkedin&utm_medium=social&utm_campaign=pdw_page_launch&utm_content=method_note_public_records`
